# Elder-FactCheck

[English README](./README.md)

`Elder-FactCheck` 是一个可复用的 Agent Skill，适用于 Codex、OpenClaw 以及其他类似的智能体平台。它专门用于帮助用户审核长辈转发的可疑内容，并生成一段更温和、更体面的回复建议，在减少误导信息传播的同时尽量维护家庭关系。

## 它能做什么

- 审核链接、截图、粘贴文本和转发文章摘要
- 将内容归类到常见的误导风险场景中
- 评估可信度、事实质量、情绪操纵风险和潜在利益导向
- 在需要时对关键说法进行进一步核查
- 生成适合回复给父母或祖辈的温和话术

这个项目面向的是一个非常真实、也非常常见的家庭沟通场景：

很多长辈转发内容并不是出于恶意，而是出于关心、提醒或表达情绪。一个真正有用的 agent，不只是判断“对不对”，还要帮助用户在不激化关系的前提下处理这些内容。

## 适合谁使用

- 想帮助父母或祖辈辨别转发内容的人
- 想构建更有社会情境理解能力的 Agent 开发者
- 希望参考一个具体、完整 skill 设计案例的 prompt 或 skill 作者

## 项目亮点

- 面向真实家庭沟通，而不只是机械地判断真假
- 同时兼顾内容核查与关系维护
- 即使原文不完整、只有截图，也能进行有限度的分析
- 明确处理“无法充分核实”这类边界情况

## 推荐使用场景

当用户出现以下需求时，适合调用这个 skill：

- 明确要求使用 `elder-factcheck`
- 发来长辈转发的可疑文章、截图或摘要，想知道是否可信
- 想礼貌地回复妈妈、爸爸、奶奶、爷爷等家庭长辈

## 示例场景

1. 奶奶转发一篇文章，说某种食物会“喂养癌细胞”。
2. 爸爸发来一张截图，说某项新政策明天就要正式实施。
3. 妈妈发来一篇家庭伦理文章，里面充满 guilt 和绝对化表达。
4. 爷爷转发一条没有官方来源的银行卡安全提醒。

## 输出结构

这个 skill 设计上会尽量输出以下内容：

- 目标沟通对象
- 内容类别
- 3 条核心观点或核心说法
- 结构化评估结果
- 综合结论
- 简短依据说明
- 核查情况
- 建议回复话术

## 设计目标

- 保护长辈的体面和善意
- 避免一上来就使用对抗式表达
- 能在证据不完整时诚实表达不确定性
- 尽量优先使用权威来源或一手来源
- 让输出对普通用户真正可用

## 局限性

- 不能替代医生、律师、监管机构、银行或其他专业人员
- 能力上仍受限于原始内容是否可访问、是否能被核查
- 当证据不足时，不应给出过度自信的判断

## 如何接入你的 Agent

你可以根据自己的平台需要调整：

- 默认沟通对象
- 不同地区的优先核查来源
- 支持的语言和地区化表达
- 面向不同家庭成员的回复语气
- 输出详细程度
